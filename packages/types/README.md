# `@workspace/types`

Bastion's on-chain types, generated by Codama from the IDL.

The Rust enum in `programs/bastion/src/state/policy.rs` is the contract. Hand-mirroring it in TypeScript drifts; we don't. `postinstall` regenerates `src/generated/` on every `pnpm install`. If you reach for `export type Foo = …` here, change the Rust enum instead.

The one hand-written symbol is `WrappedInstruction` — Codama inlines its fields onto `ExecuteInstructionData` because nothing else references it. A `Pick` in `src/index.ts` recovers the standalone shape.

## Why a separate package

So indexers, dashboards, and bots that only read chain state don't pay for the SDK's session/execute pipeline. Zero runtime weight — `@solana/kit` is a type-only peer dep. The SDK (`@workspace/sdk`) layers on top.

## Codegen

Runs on `pnpm install`. Manual: `pnpm --filter @workspace/types codegen`. After Rust changes, `anchor build` first.

## What we deliberately don't ship

| Surface | Where it belongs |
|---|---|
| Aggregate error-code map | SDK log parser |
| `PolicyKind` byte map | SDK, when memcmp filters need it |
| `Lamports` / `MicroLamports` brands | SDK public function signatures |
| `META_FLAG_*` bit constants | SDK helper that packs `{ signer, writable }` |
| `SessionSummary`, `AttachedPolicy` wrappers | Inline at the SDK callsite |

If a downstream package can't name the consumer, the abstraction isn't earning rent.
